Number Methods and Properties

#1 toString() Variable Number Method

The code below will return x as a string. The benefit in the real world would be to use this in a situation where the value is not being manipulated by math but instead needs to be used as text. 

                var x = 777;
                x.toString();

#3 toString() Expression Number Method

The code below will return the expression as a string. This could be beneficial if you needed to output a number as text, but the number needs to be processed via an expression first.

                (777 + 7000).toString();

#5 toFixed() Number Method

The code below will return z as a fixed value. This could be beneficial if there was a limit to the data type you are using, or if you wanted to calculate gas prices with great precision and return it as a fixed dollar amount.

                var z = 77.7666666;
                z.toFixed(2);

#7 valueOf() Number Method

The code below will return the variable value if it is a literal and will process an expression and return that value. If you have an object or method that needs to be a primitve value, this will do it.

                var b = 777;
                var c = 7000;
                (b + c).valueOf();

#9 MIN_VALUE Number Property

The code below will return the smallest possible number in Javascript. This could be helpful in an if statement where you need a low bottom.

                var e = Number.MIN_VALUE;

#11 NEGATIVE_INFINITY Number Property

The code below will return the overflow of -Infinity. This is helpful if you are doing the same tasks as Infinity but with negative numbers.

                var g = Number.NEGATIVE_INFINITY;

#13 isFinite() Number Method

The code below will return a true or false depending on if the number is finite, in this case, False. This could be helpful in validation. If there is a large web program using large numbers, then there may need to be validation to ensure there is not an overflow.

                var i = Number.isFinite(Infinity);

#15 isSafeInteger() Number Method

The code below will return a true or false depending on if the number is 'safe', below, it resolves as False. A safe number is any integer from (253 - 1) to -(253 - 1). I imagine this is very helpful in advanced Javscript programming.

                var k = Number.isSafeInteger(77.77);

#17 prototype Number Method

The code below will return a regular output that was developed from a custom method. You could create a lot of small simple methods or scale it to large purposes in business.

                Number.prototype.methodFourteen = function() {
                    return this.valueOf() + 3;
                };

                var m = 4;
                document.getElementById("output_17").innerHTML = m.methodFourteen();

#19 Number.EPSILON Number Property

The code below will return whether there is a difference between two numbers by incrementing from one to the next with the smallest possible incremement. I imagine this is helpful in advanced programming. Below, there is no difference between the two numbers, so it is false. To be honest, I don't totally understand EPSILON.

                var o = 1000000.1 + 0.2;
                let p = 1000000.3;
                document.getElementById("output_19").innerHTML = Math.abs(o - p) < Number.EPSILON;

#2 toString() Literal Number Method

The code below will return a literal as a string. If there is no need for a variable, you could enter in the number you need to use as text.

                (777).toString();

#4 toExponential() Number Method

The code below will return y as an exponential. This could be helpful in math applications or physics applications where they are working with large numbers that need to be truncated.

                var y = 777.66;
                y.toExponential(3);

#6 toPrecision() Number Method

The code below will return the variable as a precise number. There is a small difference between toFixed() and toPrecision(). Fixed will return the length after the decimal point while Precision will return the number's total length. In advanced programming I am sure that difference matters.

                var a = 776.6666666;
                a.toPrecision(3);

#8 MAX_VALUE Number Property

The code below will return the largest number possible in Javascript. This could be helpful if you needed a large ceiling when processing an if statement.

                var d = Number.MAX_VALUE;

#10 POSITIVE_INFINITY Number Property

The code below will return the value of Infinity. This is helpful if you needed some kind of validation if a number goes over the maximum number in Javascript.

                var f = Number.POSITIVE_INFINITY;

#12 NaN - Not a Number, Number Property

The code below will return the value of NaN. This could be helpful in an if statement where you are performing validation to see if the input is a number or not.

                var h = Number.NaN;

#14 isInteger() Number Method

The code below will return a true or false depending on if the number is an integer, in this case, False. This would be helpful if there is a form that needs this kind of validation, eg, a number for a date.

                var j = Number.isInteger(77.77);

#16 constructor Number Method

The code below will return the function that created the number's prototype. I am not sure where this would be useful. It might be helpful finding a bug in an output with an unknown function prototype.

                var l = 777;
                document.getElementById("output_16").innerHTML = l.constructor;

#18 toLocaleString() Number Method

The code below will return a value that is using a different language for its output. In the example below the number '7437734' will be converted to Arabic.

                var n = new Number(7437734).toLocaleString("ar-EG");

#20 parseInt() Number Method

The code below will return a number from a string that may contain non numbers. If a value is already text, I could see how it would be helpful to be able to convert it to an integer for math processing.

                var q = "77.77abdjk";
                Number.parseFloat(q);
